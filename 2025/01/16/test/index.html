    一、下载资源的常见下载方式

    中心化下载

    （一）HTTP/HTTPS 下载

    • 原理：文件资源存储在 Web 服务器上，客户端通过 HTTP/HTTPS 请求获取资源。
    • 特点：最常见的下载方式，操作简单；支持直接点击链接下载，但受服务器带宽限制，速度可能变慢，不支持断点续传。

    （二）FTP/SFTP 下载

    • 原理：基于文件传输协议的客户端-服务器模式，通过 FTP/SFTP 协议传输文件。
    • 特点：速度稳定，支持断点续传，适合大文件传输；但资源少，需服务器支持，下载人数多时速度可能下降。

    去中心化下载

    （一）BT 种子下载

    • 原理：通过 Tracker 服务器获取下载者 IP，下载者间互相交换文件块。
    • 特点：无需中心服务器，下载人数越多速度越快；但依赖 Tracker 服务器。

    （二）磁力下载

    • 原理：通过 DHT 网络技术，直接利用磁力链接中 infohash 信息定位资源。
    • 特点：进一步去中心化，不依赖 Tracker，稳定性强。

    （三）eD2k 下载

    • 原理：基于 eDonkey2000 网络的 P2P 文件共享模式。
    • 特点：用户既是下载者也是资源提供者，充分利用节点资源。

    基于编程接口的下载

    （一）API 下载

    • 原理：调用网站或服务提供的 API 接口，自动化获取资源。
    • 特点：灵活个性化，适合批量下载或集成到应用程序。

    （二）aria2 下载

    • 原理：支持多协议和多来源的轻量级命令行下载工具。
    • 特点：下载速度快，支持多线程和多分片，高度可定制。

    （三）RPC 下载

    • 原理：客户端通过 RPC 协议调用服务器的下载服务。
    • 特点：跨网络和平台，适合分布式下载，但需服务器端支持。

    二、Mac 上的下载资源工具

    （一）浏览器自带下载功能

    • 适用下载方式：HTTP/HTTPS
    • 特点：简单便捷，适合日常简单文件下载。

    （二）Transmit

    • 适用下载方式：FTP/SFTP
    • 特点：功能强大的 FTP 客户端，支持断点续传，界面友好。

    （三）uTorrent

    • 适用下载方式：BT 种子、磁力下载
    • 特点：流行的 BT 客户端，支持多任务并行下载。

    （四）Folx

    • 适用下载方式：HTTP/HTTPS、BT 种子、磁力下载
    • 特点：支持智能下载管理、断点续传和分类存放。

    （五）aria2c（需安装配置）

    • 适用下载方式：HTTP/HTTPS、FTP、BT 种子、磁力下载
    • 特点：命令行工具，支持多协议、高效下载；结合图形化前端更易操作。

    （六）iGetter

    • 适用下载方式：HTTP/HTTPS、FTP
    • 特点：老牌下载工具，支持断点续传和批量下载，界面简洁易用。
