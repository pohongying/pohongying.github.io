    基于Java标准库的方法

    使用 Properties 类

    • 特点：
      • Properties 类继承自 Hashtable，专门用于处理键值对配置信息.
      • 提供了 load() 方法，可以从 InputStream 或 Reader 中加载配置文件.
    • 优点：
      • 简单易用，无需引入额外的依赖库.
      • 可以方便地获取和设置配置项的值.
    • 缺点：
      • 需要手动处理文件的读取和关闭，代码较为繁琐.
      • 不支持自动的本地化和国际化.
    • 适用场景：
      • 配置文件较为简单，主要包含键值对信息.
      • 不需要根据不同的语言环境加载不同的配置文件.
    • 示例代码：
      Properties properties = new Properties();
try (InputStream in = new FileInputStream("config.properties")) {
    properties.load(in);
    String value = properties.getProperty("key");
} catch (IOException e) {
    e.printStackTrace();
}

    使用 ResourceBundle 类

    • 特点：
      • ResourceBundle 类用于加载本地化的资源文件，支持根据不同的语言环境自动加载相应的资源文件.
      • 提供了 getBundle() 方法，可以根据指定的基名和语言环境获取资源包.
    • 优点：
      • 支持国际化和本地化，可以根据用户的语言环境自动切换配置.
      • 无需指定文件扩展名，使用起来较为方便.
    • 缺点：
      • 主要用于本地化资源的加载，不适用于所有类型的配置文件.
      • 需要为每种语言环境准备相应的资源文件.
    • 适用场景：
      • 需要支持多语言界面或本地化提示信息的应用程序.
      • 配置信息需要根据不同语言环境进行调整.
    • 示例代码：
      ResourceBundle resource = ResourceBundle.getBundle("config");
String value = resource.getString("key");

    基于文件路径的方法

    使用 ClassLoader 读取配置文件

    • 特点：
      • 通过类加载器（ClassLoader）获取配置文件的输入流（InputStream）.
      • 只能读取类路径下的配置文件.
    • 优点：
      • 可以方便地读取类路径下的配置文件，适合于配置文件与应用程序打包在一起的场景.
    • 缺点：
      • 只能读取类路径下的配置文件，无法读取外部路径的文件.
    • 适用场景：
      • 应用程序的配置文件需要与代码一起打包部署.
      • 配置文件不需要频繁修改，或者修改后需要重新部署应用程序.
    • 示例代码：
      Properties properties = new Properties();
InputStream in = PropertiesMain.class.getClassLoader().getResourceAsStream("config/config.properties");
properties.load(in);
String value = properties.getProperty("key");

    使用 InputStream 读取配置文件

    • 特点：
      • 使用 FileInputStream 或 BufferedReader 从文件系统中读取配置文件.
      • 可以读取任意路径下的配置文件，包括本地文件系统和网络文件系统.
    • 优点：
      • 灵活性高，可以读取任意路径下的配置文件.
      • 适用于配置文件需要频繁修改或更新的场景.
    • 缺点：
      • 需要手动处理文件路径和输入流的关闭，代码较为繁琐.
    • 适用场景：
      • 配置文件位于外部路径，需要在运行时动态加载.
      • 配置文件需要频繁修改，且修改后不需要重新部署应用程序.
    • 示例代码：
      Properties properties = new Properties();
try (BufferedReader bufferedReader = new BufferedReader(new FileReader("E:/config.properties"))) {
    properties.load(bufferedReader);
    String value = properties.getProperty("key");
}

    基于第三方库的方法

    Apache Commons Configuration

    • 特点：
      • 使用Apache Commons Configuration库来加载和管理配置文件.
      • 支持多种配置文件格式，如 .properties, .xml, .ini 等.
    • 优点：
      • 功能强大，提供了丰富的API来处理复杂的配置需求.
      • 可以轻松切换不同的配置文件格式.
    • 缺点：
      • 需要引入额外的依赖库，增加了项目的依赖管理.
    • 适用场景：
      • 需要处理复杂配置文件或多种配置文件格式的场景.
      • 配置文件中包含复杂的数据结构或需要进行高级配置管理.
    • 示例代码：
      Configuration config = new PropertiesConfiguration("config.properties");
String value = config.getString("key");

    Jackson或Gson

    • 特点：
      • 使用Jackson或Gson库来解析JSON格式的配置文件.
      • 将JSON数据转换为Java对象，便于程序处理.
    • 优点：
      • 支持JSON格式的配置文件，符合现代应用的配置习惯.
      • 提供了强大的数据绑定功能，可以将配置数据直接映射到Java对象中.
    • 缺点：
      • 配置文件必须是JSON格式，对文件格式有一定要求.
      • 需要引入额外的依赖库.
    • 适用场景：
      • 配置文件为JSON格式，需要灵活的数据结构和强大的数据处理能力.
      • 需要将配置数据直接映射到Java对象中进行处理.
    • 示例代码（使用Jackson）：
      ObjectMapper mapper = new ObjectMapper();
Map<String, String> config = mapper.readValue(new File("config.json"), new TypeReference<Map<String, String>>(){});
String value = config.get("key");

    总结与比较

    方法类别 特点 优点 缺点 适用场景
    基于Java标准库 使用Java内置的类和方法 简单易用，无需额外依赖 功能有限，不支持复杂配置 简单配置文件读取
    基于文件路径 相对路径或绝对路径 路径灵活，易于管理 相对路径可能失效 配置文件位置固定或动态加载
    基于第三方库 支持多种格式和复杂配置 功能强大，支持复杂数据结构 需要引入额外依赖 复杂配置文件处理
